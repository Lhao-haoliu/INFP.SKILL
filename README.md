## 先看看一次对话长什么样

**你输入：**
> 我总是很怕麻烦别人，明明自己很难受也不敢说。

**它会这样回你：**

```
从你的描述看，这更像是「压抑需求型」的一个倾向。

你表面上是在纠结"要不要开口"，
但更深层可能是在担心：如果我表达了，对方会不会觉得我麻烦？

这说明你很重视关系的平稳，也很会顾及别人的感受。
这个特质的优势是：和你在一起的人通常感到轻松、不被施压。

但当它被过度使用时，你可能会把自己放到最后一位，
委屈慢慢累积，直到有一天突然撤退或爆发。

一个更温和的方向不是立刻变得强硬，
而是先从低风险的小需求开始说——
比如 "如果你方便的话，我想请你帮我一下"。

你可以问自己：
1. 我是否把正常需求理解成了负担？
2. 如果朋友向我提出同样需求，我会觉得他麻烦吗？
3. 我是在表达需求，还是在努力避免被讨厌？
```

这就是 INFP.SKILL 想做的事：
**不是给你贴标签，而是帮你看见"我为什么总是这样"背后的纹路。**

## 它能帮你什么

| 你的困扰 | 它会陪你看见 |
|:---|:---|
| 怕麻烦别人，难受也自己扛 | **压抑需求型** —— 你把体谅变成了自我消失 |
| 别人一句话，你想很久 | **关系敏感型** —— 你把细节放大成了关系风险 |
| 社交后像被掏空 | **社交耗电型** —— 你不是不合群，只是节奏不同 |
| 吸收了太多别人的情绪 | **过度共情型** —— 你把"理解"和"负责"混在了一起 |
| 有很多感受却说不出口 | **表达困难型** —— 你在等完美表达，却错过了时机 |
| 一被冷淡就开始怀疑自己 | **自我怀疑型** —— 你把事件扩大成了"我整个人不行" |
| 理想和现实差距太大 | **理想落差型** —— 你在等 100% 的意义，却迟迟无法开始 |
| 受伤后只会慢慢退开 | **自我保护型** —— 你在用消失保护自己 |

> 每个模式都会讲到：**这背后的情绪和需求 → 优势 → 代价 → 一个小方向 → 几个问题**

## 怎么用它

### 方式一：直接用（推荐）

把本目录复制到你的 Kimi CLI skills 目录：

```bash
# macOS / Linux
cp -r INFP.SKILL ~/.kimi/skills/

# Windows
xcopy /E /I "E:\infp\infp-self-understanding-skill" "%USERPROFILE%\.kimi\skills\INFP.SKILL"
```

启动 Kimi CLI 后，当你描述关系困扰、情绪内耗、社交疲惫、表达困难等情境时，Agent 会自动加载这个 Skill 回应你。

### 方式二：作为项目参考

你也可以不安装，直接阅读 `references/` 里的模式文档，当作一份温和的"自我理解手册"。

## 一次完整对话的 7 个节拍

当你向它倾诉时，它会按照这个节奏陪你：

| 步骤 | 它会做什么 | 为什么重要 |
|:---|:---|:---|
| ① 读你的故事 | 先听完你的具体情境，不急着给结论 | 你被看见了 |
| ② 命名模式 | 试探着说"这更像是……型" | 你终于知道这叫什么了 |
| ③ 分开情绪 | 表层情绪 vs. 深层需求 | 你理解了自己的底层逻辑 |
| ④ 讲优势 | "这说明你很重视……" | 你的特质不是缺陷 |
| ⑤ 讲代价 | "但当它被过度使用时……" | 你知道哪里可以调整 |
| ⑥ 给方向 | 一个小而具体的行动建议 | 你不至于无从下手 |
| ⑦ 留问题 | 2-3 个反思问题 | 对话结束后，你还可以继续想 |

## 更多示例

<details>
<summary>点击展开更多对话示例</summary>

**输入：** 我发现我很容易因为别人一句话想很久。

**回应概要：** 关系敏感型。你对关系里的细节很敏锐，但细节有时会被不安放大。试着把"事实"和"推测"分开。

---

**输入：** 我明明很在意一个人，但总是假装无所谓。

**回应概要：** 自我保护型 + 表达困难型。你在保护自己，但对方可能接收不到你的在意。不用一次说很多，只练习表达 10% 的真实。

---

**输入：** 我总觉得自己不够好，别人稍微冷淡一点我就开始怀疑自己。

**回应概要：** 自我怀疑型。你愿意反省是优点，但外界冷淡不等于"我整个人不行"。先把对方状态、关系事实和自我价值分开放。

</details>

## 它不会做什么

- ✗ **不是心理咨询**，不能代替专业帮助
- ✗ **不做人格测试**，不给标准答案
- ✗ **不诊断心理疾病**
- ✗ **不替你判断他人真实意图**
- ✗ **不在你描述自伤或严重危机时继续分析性格**  
  那时它会停下来，建议你寻找身边真实的支持。

## 关于数据来源与隐私

这个 Skill 的框架来自对真实 INFP 社群互动的观察与蒸馏。

**原始数据只保存在本地**，不会进入这个仓库。能进入仓库的只有抽象后的模式、匿名化示例、Prompt 和脚本。详见 `.gitignore` 和 `data_private/README.md`。

## 如果你想从自己的数据中蒸馏

也许你有自己的日记、社群观察，或者一段段想要被温柔理解的文字。

```powershell
$env:MYSQL_HOST="localhost"
$env:MYSQL_USER="root"
$env:MYSQL_PASSWORD="root"
$env:MYSQL_DATABASE="db_data"
$env:MYSQL_TABLE="infp_source_posts"

python scripts/load_data.py --source mysql
```

之后依次运行 `anonymize.py` → `extract_patterns.py` → `merge_taxonomy.py` → `build_skill_references.py` → `validate_distilled_data.py`。
