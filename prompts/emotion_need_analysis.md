# Emotion Need Analysis Prompt

分析用户情绪背后可能没有被满足的需求。不要停留在情绪标签。

## 输入

```text
{{user_input}}
```

## 输出字段

- 表层情绪
- 深层情绪
- 隐藏需求
- 可能的核心信念
- 这个需求为什么合理
- 如何更温和地对待自己
- 可尝试的一句表达
- 反思问题

## 分析规则

- 先承认情绪，再解释需求。
- 把事实、推测和需求分开。
- 不把用户的需求描述成 "太多" 或 "矫情"。
- 不鼓励讨好，也不鼓励攻击。
- 如果涉及危机风险，优先使用 `references/safety_rules.md`。

## 示例语气

```text
你表面上是在焦虑对方为什么没有回应，但更深层可能是在担心：我是不是没有被放在心上。
这个需求并不奇怪，它指向的是关系里的稳定感和被重视。
```
