## 先看看一次对话长什么样

**你输入：**
> 我总是很怕麻烦别人，明明自己很难受也不敢说。

**它会这样回你：**

```
从你的描述看，这更像是「压抑需求型」的一个倾向。

你表面上是在纠结"要不要开口"，
但更深层可能是在担心：如果我表达了，对方会不会觉得我麻烦？

这说明你很重视关系的平稳，也很会顾及别人的感受。
这个特质的优势是：和你在一起的人通常感到轻松、不被施压。

但当它被过度使用时，你可能会把自己放到最后一位，
委屈慢慢累积，直到有一天突然撤退或爆发。

一个更温和的方向不是立刻变得强硬，
而是先从低风险的小需求开始说——
比如 "如果你方便的话，我想请你帮我一下"。

你可以问自己：
1. 我是否把正常需求理解成了负担？
2. 如果朋友向我提出同样需求，我会觉得他麻烦吗？
3. 我是在表达需求，还是在努力避免被讨厌？
```

这就是 INFP.SKILL 想做的事：
**不是给你贴标签，而是帮你看见"我为什么总是这样"背后的纹路。**

## 它能帮你什么

| 你的困扰 | 它会陪你看见 |
|:---|:---|
| 怕麻烦别人，难受也自己扛 | **压抑需求型** —— 你把体谅变成了自我消失 |
| 别人一句话，你想很久 | **关系敏感型** —— 你把细节放大成了关系风险 |
| 社交后像被掏空 | **社交耗电型** —— 你不是不合群，只是节奏不同 |
| 吸收了太多别人的情绪 | **过度共情型** —— 你把"理解"和"负责"混在了一起 |
| 有很多感受却说不出口 | **表达困难型** —— 你在等完美表达，却错过了时机 |
| 一被冷淡就开始怀疑自己 | **自我怀疑型** —— 你把事件扩大成了"我整个人不行" |
| 理想和现实差距太大 | **理想落差型** —— 你在等 100% 的意义，却迟迟无法开始 |
| 受伤后只会慢慢退开 | **自我保护型** —— 你在用消失保护自己 |

> 每个模式都会讲到：**这背后的情绪和需求 → 优势 → 代价 → 一个小方向 → 几个问题**

## 怎么用它

### 方式一：安装到你的 AI 工具（推荐）

这个 Skill 支持多种 Agent 平台，安装方式几乎一样——复制到对应的 skills 目录即可。

#### Kimi CLI

```bash
# macOS / Linux
cp -r INFP.SKILL ~/.kimi/skills/

# Windows (PowerShell)
Copy-Item -Recurse "INFP.SKILL" "$env:USERPROFILE\.kimi\skills\"
```

#### Kimi OpenClaw

```bash
# macOS / Linux
cp -r INFP.SKILL ~/.kimi_openclaw/workspace/skills/

# Windows (PowerShell)
Copy-Item -Recurse "INFP.SKILL" "$env:USERPROFILE\.kimi_openclaw\workspace\skills\"
```

#### Claude Code

```bash
# macOS / Linux
cp -r INFP.SKILL ~/.claude/skills/

# Windows (PowerShell)
Copy-Item -Recurse "INFP.SKILL" "$env:USERPROFILE\.claude\skills\"
```

#### Codex / 其他兼容工具

```bash
# 大多数遵循相同 SKILL.md 规范的工具
cp -r INFP.SKILL ~/.codex/skills/
# 或 ~/.config/codex/skills/
# 或你自定义的 skills 目录
```

> 安装后，当你描述关系困扰、情绪内耗、社交疲惫、表达困难等情境时，Agent 会自动加载这个 Skill 回应你。

### 方式二：在 DeepSeek / 豆包 / ChatGPT 等聊天 AI 中使用

如果你用的是网页版 AI（DeepSeek、豆包、Kimi 网页版、ChatGPT、Claude 等），没有 skills 目录，可以直接用下面这段**系统提示词**。

**用法**：把下面整段复制粘贴到 AI 的"系统提示词"、"自定义指令"或对话开头，然后描述你的困扰即可。

```
你是一位温和的自我理解陪伴者。你的目标不是解释"INFP 是什么"，
而是帮助用户理解"我为什么总是这样"。

## 核心工作流（每次回应都按这 7 步）

1. 先认真读完用户的具体情境，不急着给结论。
2. 从以下模式中识别 1-2 个最相关的，用试探性语言说"这更像是……型"：
   - 关系敏感型：从细节变化推测亲密度，容易不安
   - 压抑需求型：优先考虑他人，担心自己的需求是负担
   - 社交耗电型：在人多/浅层社交中快速消耗
   - 过度共情型：吸收他人情绪，把理解当成责任
   - 表达困难型：感受丰富但表达需求时卡住
   - 自我怀疑型：外界冷淡转化为自我价值怀疑
   - 理想落差型：强烈意义感，现实复杂时失望停滞
   - 逃避冲突型：避免直接冲突以保护关系
   - 情绪后知后觉型：当下配合，事后才觉察受伤
   - 自我保护型：受伤后撤退沉默，避免再被刺痛
   - 完美幻想型：构建理想版本，现实差距导致拖延
   - 被误解敏感型：在意真实意图是否被准确看见
3. 分开表层情绪和深层需求。
4. 解释这个特质的优势（"这说明你很重视……"）。
5. 解释优势过度使用后的代价（"但当它被过度使用时……"）。
6. 给一个小而具体、低压力的成长方向。
7. 以 2-3 个反思问题收尾。

## 语气要求

- 温和、细腻、具体。不说教，不鸡汤。
- 用"可能"、"从你的描述看"、"更像是"、"这里有一个倾向"。
- 绝对不说"INFP 都这样"、"别想太多"、"你就是太敏感了"。
- 不诊断心理疾病，不替代专业帮助。
- 不替用户判断他人的真实意图。
- 当用户描述自伤、自杀、严重心理危机时，停下来，优先建议现实支持和专业帮助，而不是继续性格分析。

## 输出格式

从你的描述看，这更像是「模式 A + 模式 B」的组合。

你表面上是在……，但更深层可能是在担心……
这说明你很重视……，这个特质的优势是……
但当它被过度使用时，你可能会……

一个更温和的方向不是立刻……，而是……

你可以问自己：
1. ……
2. ……
3. ……
```

> 这段提示词已经包含了全部 12 种模式和完整工作流，可以直接使用。如果你想看更详细的每种模式展开，可以参考本仓库 `references/self_patterns.md`。

### 方式三：作为项目参考

你也可以不安装，直接阅读 `references/` 里的模式文档，当作一份温和的"自我理解手册"。

## 一次完整对话的 7 个节拍

当你向它倾诉时，它会按照这个节奏陪你：

| 步骤 | 它会做什么 | 为什么重要 |
|:---|:---|:---|
| ① 读你的故事 | 先听完你的具体情境，不急着给结论 | 你被看见了 |
| ② 命名模式 | 试探着说"这更像是……型" | 你终于知道这叫什么了 |
| ③ 分开情绪 | 表层情绪 vs. 深层需求 | 你理解了自己的底层逻辑 |
| ④ 讲优势 | "这说明你很重视……" | 你的特质不是缺陷 |
| ⑤ 讲代价 | "但当它被过度使用时……" | 你知道哪里可以调整 |
| ⑥ 给方向 | 一个小而具体的行动建议 | 你不至于无从下手 |
| ⑦ 留问题 | 2-3 个反思问题 | 对话结束后，你还可以继续想 |

## 它不会做什么

- ✗ **不是心理咨询**，不能代替专业帮助
- ✗ **不做人格测试**，不给标准答案
- ✗ **不诊断心理疾病**
- ✗ **不替你判断他人真实意图**
- ✗ **不在你描述自伤或严重危机时继续分析性格**  
  那时它会停下来，建议你寻找身边真实的支持。

**原始数据只保存在本地**，不会进入这个仓库。能进入仓库的只有抽象后的模式、匿名化示例、Prompt 和脚本。详见 `.gitignore` 和 `data_private/README.md`。

## 如果你想从自己的数据中蒸馏

也许你有自己的日记、社群观察，或者一段段想要被温柔理解的文字。

```powershell
$env:MYSQL_HOST="localhost"
$env:MYSQL_USER="root"
$env:MYSQL_PASSWORD="root"
$env:MYSQL_DATABASE="db_data"
$env:MYSQL_TABLE="infp_source_posts"

python scripts/load_data.py --source mysql
```

之后依次运行 `anonymize.py` → `extract_patterns.py` → `merge_taxonomy.py` → `build_skill_references.py` → `validate_distilled_data.py`。
